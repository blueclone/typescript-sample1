gitの備忘録

git

最頻出のgitコマンド
add, checkout(またはswitch), clone, commit, fetch, merge, pull, push
頻出のgitコマンド
reset, rebase, cherry-pick

使い方一例

$ git add .
ステージへ全ての変更を持っていくコマンド
$ git commit -m "コメント"
コメント付きでcommitする
$ git push origin HEAD
ローカルのブランチをリモートの同名ブランチにpushする

gitのマイルール

commitはタスク単位でやろう
なぜなら、大きな作業単位だと、あとで見返す時の負担が重くなるから
commitしたら、すぐpushしよう
なぜなら、もしPCが壊れたとしても、リモートに上げているので作業の回復の労力が少なくなるから
