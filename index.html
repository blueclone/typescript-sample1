フロントエンドの勉強

node.jsのnpmコマンドについて

$ npm init
「現在のディレクトリ下はnpmが管理する」という宣言
$ npm install パッケージ名
npmのリポジトリからパッケージをダウンロードしてくるコマンド
$ npm run hogehoge
package.jsonに書かれたシェルスクリプトを実行するコマンド 
例:$ npm run devはpackage.jsonのdevを実行する

gitのコマンド

よく使うgitのコマンド
add, checkout(またはswitch), clone, commit, fetch, merge, pull, push
その次に使うgitのコマンド
reset rebase cherry-pick

gitコマンドの具体例

$ git add .
変更からステージへ全てを持っていくコマンド
$ git commit -m "内容"
コミットする内容を記述する
$ git push origin HEAD
ローカルのブランチをリモートの同名ブランチにpushする

gitのマイルール

commitはタスク単位でやろう
なぜなら、大きな作業単位だと、あとで見返す時の負担が重くなるから
commitしたら、すぐpushしよう
なぜなら、もしPCが壊れたとしても、リモートに上げているので作業の回復の労力が少なくなるから
